# Gym Management Database System

A relational database system designed to centralize and manage all gym operations, including members, staff, subscriptions, sessions, and services.

## Team
- Omar Ahmed
- Mohamed
- Amr

## Tools Used
- SQL Server
- ERD Modeling

## Database Structure
The database contains 12+ tables covering:
- Member & Member_Phone
- Subscription
- MemberShip & Type_Details
- MemberShip_Service
- Service
- Session
- Staff & Staff_Phone
- Department
- Equipment

## Key Features
- Designed a full Entity Relationship Diagram (ERD) with all relationships
- Applied normalization to reduce data redundancy
- Modeled Many-to-Many relationships using junction tables
- Written queries covering SELECT, JOIN, and Aggregate Functions

## Queries Include
- Expired and active subscription tracking
- Staff salary analysis by year
- Session tracking per trainer and per service
- Service pricing statistics (Average, Max, Min)
- Member and equipment counts per department
